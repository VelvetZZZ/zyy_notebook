# 编程范式对比笔记

## Functional Programming（函数式编程）

**特点：**
- 所有函数都是纯函数（pure functions）。
- 无重新赋值操作，且不允许可变数据类型（immutable data）。
- 名称与值之间的绑定是永久性的。

**优势：**
- 表达式的值与其子表达式求值的顺序无关。
- 子表达式可以并行（parallel）或按需（lazy）计算。
- **引用透明性（Referential Transparency）**：
  替换一个表达式为其子表达式的值不会改变整个表达式的结果。

**缺点或限制：**
- 没有 `for` / `while` 语句 —— 需要使用递归处理迭代。
- 但可以使用尾递归（tail recursion）实现高效的迭代。

## 其他编程范式对比

### Imperative Programming（命令式编程）
- 程序由一系列命令构成，依次修改程序状态。
- 常用语言：C、Java、Python（部分）
- 优势：直观，适合硬件思维；容易控制流程。
- 缺点：副作用多，难以并行化，维护难度较大。

### Object-Oriented Programming（面向对象编程）
- 将数据和操作封装成对象。
- 特征：封装、继承、多态。
- 常用语言：Java、C++、Python。
- 优势：可复用性强，适合建模复杂系统。
- 缺点：复杂性高，过度设计容易导致臃肿。

### Procedural Programming（过程式编程）
- 基于过程或子程序（函数）进行组织的命令式编程。
- 常用语言：C、Pascal。
- 优势：结构化强，容易理解。
- 缺点：抽象能力较低，模块复用差。

### Logic Programming（逻辑编程）
- 基于逻辑推理和规则的声明式编程。
- 常用语言：Prolog。
- 优势：适合问题推理和知识表达。
- 缺点：执行效率不高，学习曲线陡峭。

### Declarative Programming（声明式编程）
- 描述“做什么”而不是“怎么做”。
- 包括函数式编程、逻辑编程、SQL 等。
- 优势：简洁、易并行化、可读性高。
- 缺点：抽象高，对底层控制较弱。

|--------------|---------------|----------------|---------------|----------------|

| 命令式       | 是            | 是             | 否            | C, Python       |
| 面向对象     | 是            | 是             | 否            | Java, C++       |
| 过程式       | 是            | 是             | 一般          | C, Pascal       |
| 逻辑式       | 否            | 否             | 是            | Prolog          |
| 声明式       | 否            | 否             | 是            | SQL, Haskell    |
